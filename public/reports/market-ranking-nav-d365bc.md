## 1. Executive Summary

The NAV → D365 Business Central corridor represents a **massive forced migration** driven by NAV end-of-life timelines. Unlike discretionary ERP upgrades, NAV users have no choice — Microsoft has sunset all NAV versions with extended support ending by 2028. This creates an enormous captive audience estimated at 100,000+ companies globally still running NAV/Navision.

**Key findings:**
- **Microsoft dominates informational SERP** with learn.microsoft.com appearing #1 in 15+ queries
- **Three fixed-price specialists** (NAVtoBC, Simplanova, Mercurius IT) have carved out a niche — exactly the positioning Source should attack

## 8. Unique Corridor Dynamics

### 8a. NAV End-of-Life Timeline (Critical)
| NAV Version | Extended Support End | Status |
|------------|---------------------|--------|
| NAV 2013/R2 | Jan 2024 | **EXPIRED** |
| NAV 2015 | Jan 2025 | **EXPIRED** |
| NAV 2016 | Apr 2026 | **Ending soon** |
| NAV 2017 | Jan 2027 | 10 months away |
| NAV 2018 (final) | Jan 2028 | Last version, 22 months |

### 8b. Upgrade Path Complexity
The NAV→BC upgrade is NOT a simple version bump. It requires a multi-step process:

1. **NAV (any version) → Business Central v14 on-premises** (technical upgrade)
2. **BC v14 on-premises → BC v14+ on-premises** (application upgrade)  
3. **BC on-premises → BC online/SaaS** (cloud migration using Cloud Migration Tool)

For older NAV versions (pre-2015), an additional intermediate upgrade step may be required.

### 8c. C/AL to AL Code Conversion
The single biggest technical barrier in this corridor:
- NAV customizations written in C/AL must be converted to AL extensions
- Microsoft's txt2al tool handles ~60-70% of conversion automatically
- Remaining 30-40% requires manual developer intervention
- This is why upgrades cost $100K+ for heavily customized systems
- **Source AI opportunity:** Automate more of this conversion with AI

### 8d. Licensing Model Shift
- **NAV:** Concurrent user licensing (buy X licenses, any user can log in)
- **BC:** Named user licensing (each user needs their own license)
- This often INCREASES licensing costs, especially for organizations with many occasional users
- Team Members license ($8/user/month) helps for light-access users

### 8e. Geographic Distribution
- **NAV's heritage is Scandinavian** (originally Navision from Denmark)
- Strongest installed base: Scandinavia, UK, Netherlands, Germany, North America
- Nordic-specific players: MDC Nordic, businessgo.dk, Update.se, NAV24.eu, Vasilok.nl
- UK-specific players: Tecvia, Advantage, Dynamics Square UK, TNP, Alphavima, Alberon
- European players: Simplanova (Lithuania), Mercurius IT (UK/global), AlfaPeople (global), ioi.gmbh (Germany)
