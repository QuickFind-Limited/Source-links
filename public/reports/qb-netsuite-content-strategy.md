## What Makes AI Models Cite a Page

| Pattern | HIGH AEO Citation (ScaleNorth, BrokenRubik, Protelo) | ZERO AEO Citation (Embark, most SERP guides) |
|---------|------------------------------------------------------|----------------------------------------------|
| Intent match | Headings mirror buyer prompts exactly | Broad narrative sections |
| Answer format | First 1-2 sentences give direct answer + number | Long lead-ins before answer |
| Specificity | Concrete ranges, thresholds, timelines | Generic advice, few hard numbers |
| Neutrality | Diagnostic tone, decision logic | Thinly disguised product pitch |
| Retrieval shape | Short, self-contained answer blocks | Long prose requiring context |
| Coverage | Multiple intent slices (cost, fit, timeline, risk) | One monolithic "ultimate guide" |
| Citation hooks | Named frameworks, checklists, tables, "when X then Y" | Mostly opinion text |

**Key insight**: Embark proves quality alone does NOT drive AEO. Packaging for retrieval does.

## Optimal Dual-Ranking Format (Google + AEO)

**Hub + answer-cluster model:**
- 1 pillar page (3,200-3,800 words) for Google depth
- 8-10 focused supporting pages (1,800-3,000 words) for prompt-level retrieval

**Page block pattern:**
1. H1 = exact intent phrase
2. Executive Answer (60-90 words, direct verdict)
3. Quick Numbers table (cost, timeline, risk, team effort)
4. Decision Matrix by company type
5. Step Plan with milestones
6. Failure Modes and countermeasures
7. FAQ with 8-12 prompt-style questions
8. CTA Ladder: self-serve tool → workshop → implementation

**AEO formatting rules:**
- Every H2 should be a probable AI prompt
- First paragraph under each H2 must answer the question directly
- Include at least one numeric/threshold per major section
- Keep pitch sections below 15% of page length
- Add structured data (Article, FAQPage, HowTo)
